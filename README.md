Разработать блок-схему алгоритма, написать код программы на языке высокого уровня, составить тестовые примеры исходных данных, которые охватывали бы прохождение всех ветвей алгоритма, составить описание объектных моделей кода. # Отчет по учебной практике: Разработка алгоритма подсчета положительных элементов массива

##  Описание проекта

Данный репозиторий содержит материалы по учебной (технологической) практике для направления подготовки 02.03.03 "Математическое обеспечение и администрирование информационных систем" (профиль "Разработка программного обеспечения (Full-stack разработка)").

##  Цель практики

Разработка алгоритма и программы для нахождения суммы и количества положительных элементов в одномерном массиве.

##  Тестовые примеры

Для проверки корректности работы алгоритма был разработан набор из 8 тестовых сценариев, покрывающих все ветви выполнения:

| № | Входные данные `A` | N | Ожидаемая сумма | Ожидаемое кол-во | Покрытые ветви |
|---|---------------------|---|------------------|-------------------|----------------|
| 1 | `[3, 5, 2]` | 3 | 10 | 3 | только «да» (все положительные) |
| 2 | `[-4, -1, -7]` | 3 | 0 | 0 | только «нет» (все отрицательные) |
| 3 | `[3, -1, 4, 0, -5, 2]` | 6 | 9 | 3 | обе ветви + ноль |
| 4 | `[0, 0, 0]` | 3 | 0 | 0 | ноль не считается положительным |
| 5 | `[7]` | 1 | 7 | 1 | один положительный |
| 6 | `[-3]` | 1 | 0 | 0 | один отрицательный |
| 7 | `[1.5, -2.5, 3.0, 0, 4.5]` | 5 | 9.0 | 3 | вещественные числа |
| 8 | `[]` (N=0) | 0 | — | — | обработка ошибки N ≤ 0 |

### 📈 Покрытие тестами

- ✅ Все положительные элементы
- ✅ Все отрицательные элементы
- ✅ Смешанные значения с нулями
- ✅ Граничные случаи (N=0, N=1)
- ✅ Вещественные числа
- ✅ Обработка ошибок
